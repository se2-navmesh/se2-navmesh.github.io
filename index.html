Abstract

Global navigation for ground robots in complex multi-level environments requires representations that accurately capture traversable regions while enabling efficient path planning. Point clouds and volumetric maps lack explicit surface structure, while pathfinding directly on dense triangle meshes is computationally expensive. Classical navigation meshes reduce this complexity, but assume yaw-invariant traversability and are therefore unsuitable for non-circular robots in constrained spaces.

We propose the SE(2) Navigation Mesh, a polygonal representation that encodes yaw-dependent traversability. Our method evaluates traversability using footprint masks, constructs a graph over yaw-specific layers with translational and rotational connectivity, and plans paths using an A*-String Pulling-A* strategy that optimizes robot position and heading.

Why SE(2) NavMesh?

Classical navigation meshes are effective because they replace dense geometry with connected convex polygons. However, they usually assume that a robot's traversability is independent of yaw. This is reasonable for circular agents, but too coarse for legged or rectangular ground robots.

In narrow corridors, around overhanging obstacles, or on multi-level structures, the same position can be feasible for one robot heading and infeasible for another. SE(2) NavMesh keeps the compact polygonal structure of a NavMesh while explicitly encoding which yaw channels are valid in each region.

Dense Meshes Are Expensive

Direct pathfinding on reconstructed triangle meshes preserves geometry, but the graph can be too large for efficient global planning.

2D Maps Lose Structure

Grid and elevation maps simplify planning, but struggle with overlapping floors, stairs, and complex traversable surfaces.

Yaw Matters

Non-circular robots can pass through constrained spaces only when aligned correctly, so heading must be part of the representation.

System Overview

Offline construction builds the representation from a pre-built map; online construction updates it from streaming point clouds.

Method Overview

Footprint Masks

The robot footprint is rasterized over yaw channels. Continuous-yaw masks make safe rotations explicit and preserve constrained regions that are feasible only under specific headings.

Yaw Layers

Traversable polygons are lifted into yaw-specific layers. Translational edges connect neighboring polygons within a layer, while rotational edges connect feasible yaw changes.

ASA Pathfinding

A graph search, string-pulling stage, and yaw-aware refinement provide paths that respect the robot footprint and produce smooth position-heading motion.

The pipeline begins by voxelizing the input mesh and annotating walkable terrain according to the robot's height, slope, step, and footprint constraints. Footprint masks then evaluate yaw feasibility at candidate positions. Regions that are feasible for every yaw channel become safe traversable regions, while regions feasible only for selected channels become restricted regions.

Planning operates over this layered representation. Translation edges move the robot within a fixed yaw layer, and rotation edges connect compatible yaw layers in the same region. ASA pathfinding first searches this graph, then straightens the polygon corridor and refines yaw transitions to produce smooth paths.
